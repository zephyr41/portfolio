Devil Scrap

Une clé usb qui siphonne des données en 30s chrono

Accueil Blog devil-scrap

Une simple clé usb ?

Imaginez une clé USB, discrète et banale, qui, une fois branchée, siphonne silencieusement la vie privée de l'utilisateur. En quelques secondes, elle récupère photos, messages, chaque souvenir partagé. Avec elle, les secrets les plus profonds d'une personne sont à portée de main. Et maintenant, imaginez que cette clé USB, capable de tout cela, je l’ai réellement créée.

Tout a commencé avec un projet de web scraping. Pour faire simple, le web scraping est un algorithme qui permet de récupérer massivement des données sur Internet. Imaginez que vous cherchez à acheter une nouvelle paire de baskets. Vous visitez plusieurs sites pour comparer les prix, lire les avis des clients, et trouver la meilleure offre. Cela peut prendre du temps et nécessiter beaucoup de clics.

Maintenant, imaginez qu’un outil, sans que vous le sachiez, parcourt automatiquement tous ces sites pour vous. Il récolte en quelques secondes tous les avis, prix, et disponibilités pour ce modèle de baskets. Ensuite, il trie les informations et vous présente les meilleures options disponibles.

C’est là que le web scraping entre en jeu. Il récupère des données en grande quantité sur Internet, que ce soit pour des produits, des avis, ou des tendances. Ce puissant outil travaille en coulisses, souvent sans que vous vous en rendiez compte.

Et donc, où est la vie privée dans tout ça ?

Bien maintenant que j'ai introduit le web scraping, je vais vous présenter mon projet. Pour ce projet en Python, l’objectif de base était simplement de récolter des données à partir d'un site web. Cependant, en tant que passionné de cybersécurité, je voulais explorer comment lier ce projet à la cybersécurité.

J'aime beaucoup discuter des dangers de la vie privée sur Internet. Alors en étant sur Instagram sur mon ordinateur, une idée m'est venue. Mon ordinateur utilise un gestionnaire de mots de passe, souvent intégré au navigateur (dans mon cas, Firefox). Je me suis demandé s'il était possible de récupérer les mots de passe enregistrés dans le navigateur et de les utiliser pour accéder à mes comptes, afin d'extraire des informations sensibles et privées (donc, pour faire du web scraping).

En plus du script, j'avais entendu parler de ce qu'on appelle une "rubber ducky" : une clé USB qui se fait passer pour un clavier. J'ai trouvé ce concept fascinant, et il y a longtemps que je voulais réaliser un projet avec. J'ai donc eu l'idée de créer une fausse clé USB qui, une fois branchée, récupère le gestionnaire de mots de passe, extrait les informations dans un fichier, puis utilise un script Python pour se connecter aux comptes de la victime et télécharger ses données.

Comment je m'y suis pris ?

J'ai d'abord divisé le projet en 2 étapes : comment je récolte mes données et comment j'obtiens les mots de passe ? Pour faire du web scraping sur des sites connus, cela n'a pas été une mince affaire. Par exemple, Instagram obfusque le code de la page.
class=ldazjdlkzaj (non ce n'est pas le chat qui marche sur le clavier, c'est un bout du code de la page d'Instagram)

Pour réaliser mon web scraping, j'ai utilisé Selenium & BeautifulSoup. Ce sont des outils complémentaires pour le web scraping. Beautiful Soup simplifie l'extraction et l'analyse des données HTML, tandis que Selenium automatise les interactions avec les navigateurs pour récupérer des données dynamiques. Je vais expliquer pourquoi j'ai surtout utilisé Selenium.

En fait, Selenium me permet d'avoir un navigateur qui interagit avec du code. Par exemple, je peux ouvrir une page sur Google et lui dire d'écrire dans la barre de recherche : "Quel temps fait-il ?" Juste avec du code, il est donc beaucoup plus simple de l'utiliser pour naviguer et faire des tests quand le code source est obfusqué.

Je ne fournirai pas de code source pour cet outil, car il peut être dangereux s'il est mal utilisé. Le développement a été long et fastidieux, avec environ 60 heures de travail investies. Cet outil fonctionne avec divers sites, dont GitHub, mais je tiens à rappeler que ce projet est absolument interdit et condamnable. Il a été réalisé dans un but purement éducatif, sur un compte personnel qui a été supprimé par la suite. Le risque associé à l'utilisation de cet outil peut aller jusqu'à 50 000 € d'amende et 3 ans d'emprisonnement. J'ai publié une vidéo YouTube à ce sujet, accessible au public, pour expliquer rapidement le processus tout en soulignant ces avertissements. Pour clarifier, j'ai analysé les éléments récurrents sur les pages web afin de permettre à Selenium d'interagir correctement avec eux (comme entrer le mot de passe). J'ai répété cette opération jusqu'à ce que je puisse identifier les éléments nécessaires pour se connecter, accéder aux messages privés, et les télécharger. La récupération des informations s'est ensuite faite en copiant directement le code source de la page.

Je prépare une vidéo beaucoup plus professionnelle et poussé sur cette outil (environ 15 minutes) pour détailler concretement le processus et les enjeux. Restez prudents et informés des lois en vigueur.
